Official source anchors

Use these as the rule-reference spine, not vague “TTB-style” language:

27 CFR §16.2127 CFR §16.2227 CFR §4.3227 CFR §5.6327 CFR §7.63

  • Health warning exact text and legibility live in Part 16.
  • Wine mandatory fields differ from spirits/malt.
  • Spirits require same-field-of-vision brand/class/ABV.
  • Malt alcohol content is profile-dependent; don’t hardcode it as always mandatory without rule profile.
  • Layout/font/contrast checks require evidence spans or image-region analysis; text extraction alone cannot prove them.

Extractor is low-trust

It extracts only visible text/fields. It cannot approve. Low confidence pushes review.

RuleGraph is authoritative

Commodity profiles map fields to CFR/TTB references and severity.

Human owns final call

System recommendation is separated from final reviewer disposition and override rationale.

End-to-end product flow

  1. Create/import application record.
  2. Upload labels/panels; create batch.
  3. Queue processes labels with concurrency and isolation.
  4. Extractor produces fields, confidence, notes, evidence spans.
  5. RuleGraph checks commodity/profile requirements.
  6. Discrepancy agent groups failures and drafts reviewer note.
  7. Reviewer accepts/rejects/overrides/escalates.
  8. Export audit packet or Teams-ready summary.

Batch architecture

The API accepting `labels: []` is not real batch processing. Real batch means job state, isolation, progress, retry, export, and filters.

POST /api/batches
→ { batchId, labels: [{ labelId, status: 'queued' }] }

POST /api/batches/:id/run
→ starts queue/concurrency pool

GET /api/batches/:id
→ statuses + results + aggregate counts

POST /api/batches/:id/export
→ JSON/CSV review packet; no raw image bytes by default

Prototype shortcut

If avoiding persistence, keep batch state client-side but still model label jobs explicitly:

  • `labelId`, `fileName`, `panel`, `status`
  • `extractorMode`, `elapsedMs`, `retryCount`
  • `checks[]`, `blockingCount`, `warningCount`
  • `reviewerDisposition`, `overrideReason`, `note`

This gives credible UI and export without pretending the demo is production infrastructure.
